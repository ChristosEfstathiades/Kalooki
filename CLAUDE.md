# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

Turborepo monorepo (npm workspaces, Node >= 24) built on the AdonisJS API starter kit. Two workspaces:

- `apps/backend` — AdonisJS 7 REST API (the substantial app). Auth, validation, ORM, and end-to-end type generation live here.
- `apps/frontend` — intentionally empty placeholder. Its package.json scripts are `echo` stubs; pick a framework and wire it to the backend via Tuyau (see `apps/frontend/README.md`).

## Commands

Run from the repo root (Turbo fans tasks out to workspaces):

```bash
npm run dev        # turbo dev — starts backend (and frontend once configured)
npm run build      # turbo build
npm run test       # turbo test
npm run lint       # turbo lint
npm run typecheck  # turbo typecheck
npm run format     # turbo format (prettier)
```

Backend-specific (run inside `apps/backend`) — everything is driven by the AdonisJS `ace` CLI:

```bash
node ace serve --hmr          # dev server with hot reload, http://localhost:3333
node ace test                 # run all test suites
node ace test unit            # run only the "unit" suite
node ace test functional      # run only the "functional" suite
node ace test --files="tests/functional/foo.spec.ts"   # single test file
node ace build                # production build
node ace migration:run        # run DB migrations (also regenerates database/schema.ts)
node ace make:controller Name # scaffolding; see `node ace list` for all generators
tsc --noEmit                  # typecheck
eslint .                      # lint
```

## Backend architecture

Standard AdonisJS layout under `apps/backend`, with three code-generation systems layered on top. Understanding what is generated vs. hand-written is essential.

### Code generation (do not hand-edit generated files)

Generation runs via AdonisJS init hooks (`adonisrc.ts` `hooks.init`) and on migration:

- **`database/schema.ts`** — regenerated by `node ace migration:run`. Contains a `BaseModel` subclass per table (e.g. `UserSchema`, `AuthAccessTokenSchema`) with all `@column` definitions. Application models extend these; e.g. `app/models/user.ts` does `class User extends compose(UserSchema, withAuthFinder(hash))`. Add columns via migrations, not by editing models.
- **`.adonisjs/server/controllers.ts`** — auto-discovered map of controller name → dynamic import, generated from `app/controllers`. Routes reference controllers through this via `import { controllers } from '#generated/controllers'` rather than importing controller classes directly.
- **`.adonisjs/client/`** — Tuyau registry (`registry/`), route types, and transformer data types (`data.d.ts`). This is the type-safe contract the frontend consumes. Exposed to consumers through the backend package's `exports` map (`@api-starter-kit/backend/registry` and `/data`). Regenerated on dev/build.

### Request flow

Routes (`start/routes.ts`) are versioned under `/api/v1`, split into an `auth` group (signup, login) and an authenticated `account` group guarded by `middleware.auth()`. Middleware stacks are wired in `start/kernel.ts` — note `force_json_response_middleware` (forces JSON) and `silent_auth_middleware` (runs on every routed request) in the global stack.

### Response serialization

All API responses go through the custom serializer in `providers/api_provider.ts`, which augments `HttpContext` with a `serialize()` method. Controllers return `serialize({...})`; it wraps output under a `data` key and normalizes Lucid pagination metadata. Use `ctx.serialize(...)` (or `serialize.withoutWrapping(...)`) in controllers rather than returning raw objects.

### Transformers

`app/transformers/*` (e.g. `UserTransformer extends BaseTransformer<User>`) shape a model into its API representation via `toObject()` / `this.pick(...)`. Controllers call `UserTransformer.transform(model)` before serializing. Transformer output types flow into the generated `data.d.ts` for the frontend.

### Auth

Two guards configured in `config/auth.ts`: `api` (token-based, the default; uses `DbAccessTokensProvider` on the `User` model) and `web` (session-based). Switch strategies by changing the guard used in middleware. Access tokens are created via `User.accessTokens.create(user)`.

### Validation

VineJS validators in `app/validators/*`. Controllers call `request.validateUsing(someValidator)`. `signupValidator` demonstrates DB-level rules (`.unique({ table, column })`) and cross-field rules (`.sameAs`).

## Testing

Japa test runner. Two suites defined in `adonisrc.ts`: `unit` (`tests/unit/**/*.spec.ts`, 2s timeout) and `functional` (`tests/functional/**/*.spec.ts`, 30s timeout — starts an HTTP server). Plugins (assert, api-client, db assertions, auth/session api clients) are configured in `tests/bootstrap.ts`. The `test` Turbo task depends on `build`.

## Database

Lucid ORM. Default driver is SQLite via `better-sqlite3` (an allowed native build in `pnpm-workspace.yaml`; also declared under `onlyBuiltDependencies`). Environment is validated in `start/env.ts` — copy `apps/backend/.env.example` to `.env` before running; `APP_KEY` is required.
